💡 Addendum: Potential Problems and Solutions

  1. Problem: Server returns a "404 Not Found" error for the form submission.
    Cause: The form's action attribute does not match the route in the server code.
    Solution: Ensure the action matches the route in server.js, e.g., /ITC505/lab-7.
  2. Problem: CSS and JavaScript files are not loading after deployment.
    Cause: Static files are not being served correctly by Express.
    Solution: Make sure the public directory is configured in the server using express.static.
  3. Problem: Form fields keep resetting when the page is reloaded.
    Cause: No persistence is implemented to save form data.
    Solution: Implement a session or database to retain inputs if required for the assignment.
  4. Problem: Deployment to Render fails due to a missing package.json or incorrect start command.
    Solution: Ensure your repository contains a package.json with a valid start script (e.g., "start": "node server.js").
